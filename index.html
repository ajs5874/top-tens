Simple & reliable deployment.

Assuming you've already ticked the 'version control' box, the next logical step is to ensure the deployment of your code is done in a similarly efficient way. This is where put.sh comes in. Yes, we know there are other deployment solutions out there, but none of them quite worked in the way we wanted - which is why we decided to build our own.

Simplicity, reliability, speed and cost were the four key elements we used as the foundations to build put.sh on.

Version Control + Deployment.

We believe that version control is a key element in every developers toolkit. If you don't have a solution in place it's probably time you asked yourself why that is... There are some fantastic services out there - Gitlab, Github, Bitbucket and Subversion to name just a few, so you really have no excuse.
